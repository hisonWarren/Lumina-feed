# 证据可信性 · 多角色评议总卷
### Lumina Feed 深化议题：AI 总结的「可信 + 可验」

> 缘起：M1–M5 已落地（多源检索入库 / 合法 OA 全文 / AI 总结 / 调度推送，各自行为门通过）。但对一个文献工具，**最大的风险不是少功能，而是 AI 总结不可信**——若编造一个效应量 / p 值 / 样本量，使用者可能据此做出临床或科研决策，危害甚于无工具。M4 当前仅有「依据徽章 + prompt 护栏」，相对「AI 参谋不裁判」这条红线仍单薄。本卷专门评议「证据可信性」，发现需求 → 收敛 → 给出可实装设计。

---

## 0. 评议小组

| 角色 | 立场 / 关切 |
|---|---|
| **周 · 系统综述员（U3）** | 严谨、可复现、可审计；「AI 绝不能替我判断纳入/排除」；要能追溯每句话的出处 |
| **Dr. Chen · 临床医生**（新） | 会照着 TL;DR 行动；最怕被幻觉的效应量误导；要置信信号 + 撤稿警示 + 「把那句原文给我看」 |
| **U2 · 实验科学家（preprint 追踪者）** | 要快、不要仪式感；但同意「编造数字」零容忍 |
| **R-ML · 总结/检索工程师** | 谈可行性：grounding 用检索、蕴含用 LLM-judge、成本几何；提醒**校验器自己也会幻觉** |
| **Kai · 工程边界** | 别镀金；要诚实信号而非虚假精度；一个迭代能上什么；**确定性优先** |
| **Lin · 法务/伦理** | 「参谋不裁判」红线；总结误导的责任；留痕/可审计；存储 grounding 片段的版权（短引用，最小化） |

## 1. 第一轮：各角色暴露的担忧

**Dr. Chen（临床）**：我看的是那句 TL;DR。如果它写「死亡率下降 25%」，而原文其实是「无统计学差异」，我可能就误判了。我要三件事：① 点一下能看到**支撑这句话的原文片段**；② 一个**置信信号**告诉我这条总结有多「贴原文」；③ 如果文章被**撤稿/更正**，必须最显眼地告诉我。

**周（系统综述）**：可信不只是给我看片段，还要**可审计**——半年后我要能复现「这条总结基于哪段文本、哪个模型、哪天生成」。而且红线不能破：系统可以**标记**某句话没依据，但**绝不能替我删改或裁定**这篇文献要不要纳入——暴露给我，我来判断。

**R-ML（工程）**：技术上能做 claim 级 grounding——把总结切成句子，每句去源文本里找最相似的片段。但有两个坑：① **纯词面相似**会漏掉改写（paraphrase）；② 想用 **LLM 做蕴含判断**，可它**自己也可能幻觉**，你拿一个会幻觉的东西去验另一个会幻觉的东西，不靠谱。**唯一能确定性验证的是数字**——效应量、p 值、CI、样本量，这些在源文里要么有要么没有，可以精确比对。这恰好是最危险的幻觉类型。

**U2（实验）**：别让我每条都点开看片段，太累。可信信号要**默认安静、需要时展开**。但数字这事我同意——编造数字不可接受。

**Kai（边界）**：所以**确定性优先**：数字核验 + 词面/片段 grounding 一定跑（不花 LLM 调用）；LLM 蕴含校验做成**可选、默认关**，省成本也避免「以幻验幻」。置信别给假的「87%」——给个**可解释的「grounded 比例」**（几句话里几句找到了原文依据）。

**Lin（法务）**：存 grounding 片段＝存了原文短句，注意版权。只存**字符偏移 + 必要的短引用**，不要把全文再存一份分发。还有责任面：系统的角色是「**提示存疑**」，不是「保证正确」——UI 文案要诚实。「参谋不裁判」也延伸到这里：**只暴露风险，不自动改写**总结（自动「修正」会让用户以为系统在替他判断证据）。

## 2. 第二轮：收敛与冲突调和

**冲突 A：grounding 要不要 LLM？**
- R-ML：纯词面会漏 paraphrase；Kai：LLM-judge 会「以幻验幻」且费钱。
- **调和**：**两层**。第一层确定性永远跑——(a) 数字保真核验，(b) 词面/n-gram 片段匹配给 grounded 分。第二层 LLM 蕴含校验**可选**，仅用于「词面弱但可能蕴含」的句子，把它从「弱」升到「支持/不支持」；且校验器被约束**只判断蕴含、不补充事实**，输出 supported/unsupported/partial 三态。LLM-judge 永远**不能推翻**确定性的数字核验失败。

**冲突 B：发现幻觉怎么处理？**
- Dr. Chen 想要「干净的正确总结」；周/Lin 坚持「不许替我改」。
- **调和（ADR-T2）**：**暴露而非改写**。不支持的句子**保留原样 + 加显眼标记**（如 `⚠ 未在原文找到依据`），并把 grounded 比例摆出来。系统从不删改、不裁定。Dr. Chen 接受：标记够显眼即可。

**冲突 C：置信怎么表达？**
- 不给虚假百分比。**grounded 比例 + 每句状态（grounded / 弱 / 不支持 / 数字存疑）** 是可解释的具体信号。低于阈值的总结整体打「⚠ 低 grounding」横幅。

**冲突 D：成本 / 摩擦**
- 确定性层零 LLM 调用、可缓存（写 M1 summaries 旁）；LLM 校验默认关、按需开。UI 默认只显示徽章 + 比例，片段点开才看。

**红线复述（全员同意）**：① 数字幻觉零容忍（确定性核验）；② 暴露不改写（AI 不裁判延伸）；③ 撤稿/更正最显眼；④ 可审计（出处+模型+时间）；⑤ 最小留痕（偏移+短引用，不再分发全文）；⑥ 置信用具体信号，不用假精度。

## 4. ADR

- **ADR-T1 · 确定性优先**：数字核验 + 词面/片段 grounding **永远跑**（零 LLM 调用、可缓存）；LLM 蕴含校验**可选、默认关**，仅升级「弱」句，且**不得推翻**数字核验失败。理由：避免「以幻验幻」；数字是最危险且可确定性核验的部分。
- **ADR-T2 · 暴露而非改写**：只标记、不删改、不裁定。理由：「AI 参谋不裁判」延伸到证据层；用户拥有证据判断权。
- **ADR-T3 · 最小留痕**：只存字符偏移 + 短引用（默认 ≤160 字）。理由：版权 + 隐私；grounding 不是全文再分发。
- **ADR-T4 · 具体不确定性信号**：grounded 比例 + 每句状态，**不用虚假置信百分比**。理由：诚实优于假精度。
- **ADR-T5 · 撤稿优先级最高**：撤稿/更正在所有 UI 出口前置且不可折叠。理由：撤稿文献的总结无论多准都危险。

## 5. 设计（本次深化实装）

```
SummaryResult(M4) + 实际所依据的源文本(abstract|fulltext)
        │
   segment: 切句(中英) + 抽数字/统计量
        │
   ground(确定性, 永远跑):
     · 每句 → 源文句/窗口 token-overlap & n-gram 相似 → 最佳片段(偏移)+分
     · 数字核验: 句中每个数字是否在源文出现 → numbersOk
     · 状态: grounded(分≥hi) / weak(lo≤分<hi) / unsupported(分<lo)
              数字不符 → 强制降级 + 标「数字存疑」
        │
   verify(可选, LLM-judge, 仅对 weak 句):
     · 蕴含三态 supported/partial/unsupported；不得推翻数字失败
        │
   GroundedSummary:
     · claims[{text, status, score, span{start,end,quote}, numbersOk}]
     · groundedRatio, flagged[], retractedWarning
     · annotate(text): 不支持/存疑句保留原样 + 显眼标记（不改写）
        │
   审计留痕(可选): 写 summaries 旁，记 模型/时间/偏移/短引用
```

**文件计划** `src/core/trust/`：
- `segment.ts`：中英切句；抽数字与统计量（含 p<0.05、95% CI、HR 0.75、n=1200、25%）。
- `grounding.ts`：句↔源片段相似（token Jaccard + 3-gram 重叠）+ 数字核验 → 每句状态/偏移/分。
- `verifier.ts`：可选 LLM 蕴含校验（注入，三态；约束只判蕴含；不推翻数字）。
- `grounded-summary.ts`：编排 → `GroundedSummary` + `annotate` 渲染 + 比例/横幅；撤稿前置。
- `audit.ts`：最小留痕（偏移+短引用）写库接口。
- `index.ts` + `summarizeGrounded()`（包 M4 `summarizePaper`，对实际所用文本做 grounding）。

**接点**：M4 `SummaryResult` 增补 `grounded?: GroundedSummary`；`digest-glue` 在生成 tldr 后顺带 ground，卡片徽章升级为「grounded x/y · 可点看出处」；撤稿横幅置顶。

## 6. 落地顺序

1. **本卷实装（深化切片）**：T-1/T-2/T-3/T-4/T-5 + T-7（可选层）+ T-8 + T-9 接点 + T-6 留痕接口。确定性可在沙箱真跑验证；LLM 校验用注入假实现验证。
2. 后续：T-10 评测集（构造幻觉样本回归）、T-11 出处导出、live 撤稿源（Crossref `update-to` / RetractionWatch）接入 T-5 接口。

> 一句话：把「依据徽章」从**一个标签**升级为**一套可验证的证据链**——每句话可定位原文、每个数字被核验、幻觉被显眼暴露而非偷偷改写、可信度用「grounded 比例」这种具体信号表达。这是「AI 参谋不裁判」在证据层最硬的兑现。
